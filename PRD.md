# MarkFlow — 极简 Markdown 编辑器 PRD

---

## 1. 产品概述

### 一句话定位
MarkFlow 是一款面向写作者和知识工作者的苹果风格极简 Markdown 编辑器，在视觉克制与功能完整之间取得平衡。

### 目标用户
- 习惯 Markdown 写作的开发者、写作者
- 需要结构化文档管理的学生和研究人员
- 追求干净、无干扰写作环境的知识工作者
- 追求苹果设计语言的用户

### 核心价值
- **沉浸**：无干扰的写作界面，内容优先
- **结构**：文档大纲导航，快速定位
- **所见即所得**：实时预览，分屏可调
- **极简优雅**：视觉克制、大量留白、流畅动效
- **本地优先**：所有数据存储在本地 IndexedDB，无后端依赖

## 2. 功能需求

### 2.1 编辑体验

#### 2.1.1 分屏模式

| 项目 | 说明 |
|------|------|
| **输入** | 用户拖拽分屏分隔条，或点击模式切换按钮 |
| **输出** | 界面在三种模式间切换：分屏（编辑50% / 预览50%）、纯编辑（100%）、纯预览（100%） |
| **交互逻辑** | 分屏模式下，拖拽分隔条实时调整左右比例，范围 20%~80%；模式切换按钮为工具栏右侧三个图标，点击循环切换 |

#### 2.1.2 Markdown 语法高亮

| 项目 | 说明 |
|------|------|
| **输入** | 用户在编辑区输入 Markdown 文本 |
| **输出** | CodeMirror 6 编辑器内实时渲染语法高亮（标题、加粗、斜体、链接、行内代码等） |
| **配色** | 高亮色基于编辑器主题自动适配（浅色/深色），与预览区视觉一致 |

#### 2.1.3 滚动同步

| 项目 | 说明 |
|------|------|
| **输入** | 用户在编辑区或预览区滚动 |
| **输出** | 另一区域自动滚动至对应位置 |
| **交互逻辑** | 基于光标位置计算预览区对应 DOM 节点，或基于滚动百分比映射；允许用户临时解除同步（拖拽预览区滚动条时） |

#### 2.1.4 搜索与替换

| 项目 | 说明 |
|------|------|
| **输入** | Cmd/Ctrl+F 唤起搜索栏，Cmd/Ctrl+H 唤起替换栏 |
| **输出** | 搜索高亮（所有匹配项）、当前匹配项标记、匹配计数 |
| **交互逻辑** | 支持正则表达式；Enter 跳至下一匹配；Escape 关闭搜索栏；替换时确认逐个或全部替换 |
| **快捷键** | `Cmd/Ctrl+F` 搜索，`Cmd/Ctrl+H` 替换，`Cmd/Ctrl+Shift+H` 全部替换 |

## 3. 非功能需求

### 3.1 性能指标

| 指标 | 要求 |
|------|------|
| **首屏加载** | 白屏时间 < 1.5s（Vite 构建后 gzip ~150KB） |
| **输入延迟** | 键盘输入到屏幕显示 < 50ms（CodeMirror 6 目标） |
| **预览刷新** | 编辑区内容变化到预览区更新 < 100ms |
| **文件切换** | 切换文件并渲染完成 < 200ms |
| **大文档** | 支持单文档最大 10 万字，滚动/搜索无明显卡顿 |
| **内存占用** | 正常使用 < 200MB 内存 |

### 3.2 兼容性

| 项目 | 要求 |
|------|------|
| **浏览器** | Chrome 90+、Firefox 90+、Safari 15+、Edge 90+ |
| **桌面端** | 未来通过 Tauri 打包 macOS/Windows/Linux 原生应用 |
| **响应式** | 最小支持宽度 800px（平板横屏）；移动端暂不优先支持 |

### 3.3 无障碍

| 项目 | 要求 |
|------|------|
| **键盘导航** | 所有功能可通过键盘完成（Tab 切换焦点，Enter 确认，Escape 取消） |
| **ARIA 标签** | 所有图标按钮、面板、弹窗均标注 `aria-label` |
| **焦点管理** | 模态框/搜索栏打开时焦点 trapping，关闭时还原焦点 |
| **屏幕阅读器** | 支持 ARIA live regions 播报搜索结果数、文件操作结果等 |
| **字体大小** | 默认 16px，可通过浏览器缩放，不破坏布局 |

### 3.4 安全

| 项目 | 要求 |
|------|------|
| **沙箱** | 纯前端应用，无网络请求（除非用户主动插入外链图片） |
| **CSP** | 部署时可配置严格 Content-Security-Policy |
| **XSS** | 渲染用户 Markdown 时进行 HTML 转义；Mermaid 图表和 KaTeX 渲染在沙箱化 DOM 中执行 |
| **本地数据** | IndexedDB 数据仅存储在用户本地，不上传任何服务器 |

## 4. 信息架构

### 4.1 页面结构

```
┌─────────────────────────────────────────────────────────┐
│                      工具栏 (48px)                       │
├────────────┬──────────────────────────┬─────────────────┤
│            │                          │                 │
│  文件树     │        编辑区             │     大纲         │
│  (240px)   │     (可变宽度)            │    (200px)      │
│  可折叠     │                          │    可折叠        │
│            │                          │                 │
│            ├──────────────────────────┤                 │
│            │        预览区             │                 │
│            │     (分屏模式下)           │                 │
│            │                          │                 │
└────────────┴──────────────────────────┴─────────────────┘
```

### 4.2 导航逻辑

- **文件树 ↔ 编辑区**：点击文件 → 加载内容至编辑区；拖拽 → 移动文件
- **编辑区 ↔ 预览区**：实时同步；分屏模式下滚动联动
- **大纲面板**：从编辑区内容自动生成；点击 → 跳转编辑区对应位置
- **工具栏**：所有操作均触发编辑区状态变化或界面切换
- **导出菜单**：点击工具栏导出按钮 → 下拉菜单 → 选择格式 → 生成文件或复制

### 4.3 状态管理

```
AppState
├── theme: 'light' | 'dark' | 'system'
├── sidebarOpen: boolean          // 文件树开关
├── outlineOpen: boolean          // 大纲面板开关
├── viewMode: 'split' | 'edit' | 'preview'  // 视图模式
├── splitRatio: number            // 分屏比例 0.2~0.8
├── expandedFolders: Set<id>     // 文件夹展开状态
└── selectedFileId: string | null

EditorState (CodeMirror)
├── doc: string
├── cursor: { line, ch }
├── selection: { from, to }
└── search: { query, replacements, currentMatch }

FileSystemState
├── root: FileNode
└── openFiles: FileId[]

// FileNode
{
  id: string,
  type: 'file' | 'folder',
  name: string,
  parentId: string | null,
  order: number,
  content?: string,        // file 类型
  children?: FileId[]      // folder 类型
}
```

- 页面 unload 前同步保存（`beforeunload` 事件）

## 6. 数据模型

### 6.1 IndexedDB Schema

使用 `idb-keyval` 或 `localForage` 封装，底层使用两个 ObjectStore：

#### Store: `files`

| 键 | 类型 | 说明 |
|----|------|------|
| `id` (主键) | string (UUID) | 文件唯一标识 |
| `name` | string | 显示名称，含扩展名 |
| `type` | `'file'` | 节点类型 |
| `parentId` | string \| null | 父文件夹 ID，null 为根 |
| `order` | number | 同级排序权重 |
| `content` | string | Markdown 文本内容 |
| `createdAt` | number | Unix timestamp |
| `updatedAt` | number | Unix timestamp |

#### Store: `folders`

| 键 | 类型 | 说明 |
|----|------|------|
| `id` (主键) | string (UUID) | 文件夹唯一标识 |
| `name` | string | 显示名称 |
| `type` | `'folder'` | 节点类型 |
| `parentId` | string \| null | 父文件夹 ID，null 为根 |
| `order` | number | 同级排序权重 |
| `createdAt` | number | Unix timestamp |
| `updatedAt` | number | Unix timestamp |

#### Store: `blobs`

| 键 | 类型 | 说明 |
|----|------|------|
| `id` (主键) | string (UUID) | blob 唯一标识 |
| `file` | Blob | 图片二进制数据 |
| `mimeType` | string | MIME 类型 |
| `name` | string | 原文件名 |

#### Store: `settings`

| 键 | 类型 | 说明 |
|----|------|------|
| `theme` | `'light'` \| `'dark'` \| `'system'` | 主题偏好 |
| `sidebarWidth` | number | 文件树面板宽度 (px) |
| `outlineWidth` | number | 大纲面板宽度 (px) |
| `splitRatio` | number | 分屏比例 |
| `expandedFolders` | string[] | 展开的文件夹 ID 列表 |
| `lastOpenedFileId` | string \| null | 上次打开的文件 ID |
| `recentFiles` | string[] | 最近打开的文件 ID 数组 (最多 20) |

### 6.2 文件树数据结构（内存中）

```typescript
interface FileTreeNode {
  id: string;
  name: string;
  type: 'file' | 'folder';
  parentId: string | null;
  order: number;
  children?: FileTreeNode[]; // 仅 folder 有
}

type FileTree = FileTreeNode[];
```

- 内存中维护树形结构，CRUD 操作同步写 IndexedDB
- 文件树排序按 `order` 字段升序

## 7. 技术选型

### 7.1 框架与构建

| 选型 | 决定 | 备选 | 理由 |
|------|------|------|------|
| **React 版本** | React 18+ | Preact | React 18 的并发渲染对复杂编辑器体验更好 |

| **包管理** | npm | pnpm, yarn | npm 生态最全，Vite 团队默认使用 |

### 7.2 编辑器内核

| 选型 | 决定 | 备选 | 理由 |
|------|------|------|------|
| **编辑器** | CodeMirror 6 | Monaco, ProseMirror, textarea+自定义 | CodeMirror 6 轻量、模块化好、React 集成成熟；Monaco 太重（VS Code 级别）；ProseMirror 学习曲线陡峭；textarea 无法满足语法高亮和精确光标控制 |

### 7.3 Markdown 解析与渲染

| 选型 | 决定 | 备选 | 理由 |
|------|------|------|------|
| **Markdown 解析** | markdown-it + 插件 | marked, remark, markdown-it | markdown-it 插件生态完整，GFM 表格、任务列表、Mermaid 代码块均可通过插件支持 |
| **GFM 扩展** | markdown-it-task-lists, markdown-it-table | — | 官方 GFM 插件 |
| **代码高亮** | highlight.js 或 Prism | — | 内嵌代码块语法高亮，轻量且支持 100+ 语言 |
| **数学公式** | KaTeX | MathJax | KaTeX 比 MathJax 快 10 倍，渲染质量相当，无需网络加载 |
| **Mermaid** | mermaid | — | 最成熟的浏览器图表库，支持所有主流图类型 |

### 7.4 存储

| 选型 | 决定 | 备选 | 理由 |
|------|------|------|------|
| **本地存储** | idb-keyval | localForage, Dexie | idb-keyval 极简 API，足够本项目需求；localForage 太重；Dexie 过于强大 |
| **Blob 存储** | 独立 ObjectStore (原生 IndexedDB API) | — | idb-keyval 不支持 Blob，需单独处理 |

### 7.5 样式方案

| 选型 | 决定 | 备选 | 理由 |
|------|------|------|------|
| **CSS 方案** | 纯 CSS + CSS Variables | Tailwind, CSS-in-JS, Styled Components | 手写 CSS 保证完全控制，符合"不使用第三方 UI 库"要求；CSS Variables 支持主题切换 |
| **图标** | Lucide React | Heroicons, Feather Icons | Lucide 图标简洁、风格统一、React 支持好 |
| **字体** | 系统字体栈 | Google Fonts | 系统字体栈确保苹果原味，无需加载 |

### 7.6 导出相关

| 选型 | 决定 | 备选 | 理由 |
|------|------|------|------|
| **PDF** | `window.print()` | html2pdf.js | 浏览器原生打印，无需引入体积大的库，打印质量最高 |
| **HTML 导出** | 手动拼接 HTML 字符串 + 内联 CSS | — | 完全掌控导出格式，离线可用 |
| **剪贴板** | `navigator.clipboard.write()` | execCommand | 现代 API，支持富文本复制 |

### 9.2 技术风险

| 风险 | 影响 | 缓解方案 |
|------|------|----------|
| CodeMirror 6 自定义主题与 Markdown 语法高亮配置复杂 | 中 | 先行使用官方示例主题，后续按需定制 |

| Mermaid SSR/TSM 渲染性能（大型图表）| 中 | 图表懒渲染（仅视口内渲染），提供加载状态 |
| 大文档（>5 万字）滚动性能 | 中 | 分段渲染预览区虚拟列表，CodeMirror 本身无问题 |
